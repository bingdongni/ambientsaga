# AmbientSaga 境世 — 完整项目设计

## 一、项目定位

**AmbientSaga: A Decentralized Perception-Driven Multi-Agent Simulation Engine for Social-Ecological Co-Evolution**

境世：面向社会-生态共生涌现的去中心化感知驱动多智能体仿真引擎

### 1.1 核心定位

它不是"AI社交网络"，不是"AI村庄对话Demo"，而是一个：
- 面向成千上万个智能体的**AI自然世界与AI社会世界共演化平台**
- 面向社会科学、复杂性科学、AI安全研究的**可实验仿真引擎**
- 面向复杂系统研究的**大规模多智能体基准平台**

### 1.2 四个"成立"标准

| 标准 | 含义 |
|------|------|
| **规模成立** | 世界里长期存在成千上万个个体 |
| **机制成立** | 个体在资源、制度、风险、利益约束下行动 |
| **历史成立** | 世界积累记忆、组织、法律、文化、冲突史 |
| **实验成立** | 可改变量、做干预、导指标、发论文 |

### 1.3 与现有工作的关系

```
现有前沿：
- Stanford Generative Agents (25 agents, 小镇)
- AgentSociety (10k agents, 社会议题实验)
- OASIS (百万级, 社交媒体仿真)
- Concordia (生成式社会模拟框架)

AmbientSaga 的升级：
不是让更多 agent 聊天，而是构建
"自然-生态-社会-制度-文明"五层联动的文明级仿真
```

## 二、核心范式：去中心化感知驱动

这是项目区别于所有现有模拟器的**根本性范式差异**。

### 2.1 中心化架构的核心问题

现有所有主流生成式智能体项目（包括 Stanford Smallville、AI Town），均采用中心化上帝视角：
- 智能体可直接读取全局世界状态
- 中心调度器统一触发所有行为
- 这种架构存在根本性的"仿真偏差"，无法模拟真实社会的分布式决策逻辑

### 2.2 三条范式铁则

**铁则一：无全局权限**
所有智能体无任何全局世界状态读取权限，唯一信息输入是自身感知范围内的**氛围信号**。

**铁则二：无中心化调度**
无任何上帝视角的指令下发，智能体行为仅由"自身心智模型+局部感知信号"自主触发。

**铁则三：信号物理约束**
所有氛围信号严格遵循传播范围、强度衰减、持续时长的物理规律。完全复刻真实世界的感知边界——你看不到远处发生的事情。

### 2.3 氛围信号模型

```
事件发生 → 产生氛围信号 → 广播到信号总线 → 
→ 仅订阅了该信号类型的智能体被触发 
→ 智能体感知到信号，结合自身记忆做决策 
→ 执行行动，产生新事件 → 新信号

信号属性：
- 类型：资源/威胁/机会/社会/环境
- 强度：0-100，随距离衰减
- 范围：智能体可感知的最大半径
- 持续：从事件发生到信号消散的时长
- 优先级：高/中/低，影响调度顺序
```

### 2.4 涌现性的来源

正是在这三条铁则的约束下，涌现性才真正成为可能：
- 智能体无法看到全局，只能根据局部信息做决策
- 大量局部决策的汇聚，产生全局秩序
- 信息在传播中失真、放大、扭曲
- 与自然世界耦合：灾害迫使迁徙，迁徙引发冲突

## 三、四层世界模型

### 3.1 模型总览

```
┌─────────────────────────────────────────────────┐
│                  历史世界层                       │
│   历史档案 | 编年史 | 事件回放 | 因果追溯        │
├─────────────────────────────────────────────────┤
│                  社会世界层                       │
│   经济 | 政治 | 文化 | 语言 | 制度 | 组织        │
├─────────────────────────────────────────────────┤
│                  制度世界层                       │
│   法律 | 产权 | 规则 | 执法 | 合约 | 边界        │
├─────────────────────────────────────────────────┤
│                  自然世界层                       │
│   地形 | 气候 | 生态 | 资源 | 灾害 | 空间        │
└─────────────────────────────────────────────────┘
                          ↑
              双向耦合因果链
                          ↓
```

### 3.2 自然世界层（物理宇宙）

**地理**：河流、森林、平原、矿区、海岸、山地、岛屿
- 1024x1024 离散格子，每格 1km²
- 多尺度 Perlin 噪声 + 板块构造模拟
- 气候带：热带/亚热带/温带/寒带/极地

**生态**：
- 物种：动物（食草/食肉）、植物（作物/野生）
- 食物链：能量传递效率约 10%
- 演化机制：突变 + 选择压力下的物种特征漂移

**资源**：
- 基础资源：矿石、木材、石料、水
- 战略资源：铁、铜、盐、煤炭
- 农业资源：耕地、牧场、渔场
- 资源会因开采而枯竭，因生态修复而再生

**灾害**：洪水、干旱、地震、瘟疫、蝗灾、森林火灾

**关键创新：自然世界与社会世界的闭环因果链**

```
生态→社会链路：
  环境变化(旱灾/矿脉枯竭/森林减少)
    → 资源可用性下降
    → 生存约束收紧
    → 群体行为模式被迫改变(迁徙/争夺)
    → 社会结构适应性重构(新组织/新制度)

社会→生态链路：
  智能体集体行为(开荒/砍伐/建造)
    → 环境参数被累积改变
    → 生态系统稳态被打破
    → 资源分布和气候模式长期变化
    → 反向约束社会演化方向
```

### 3.3 制度世界层（规则宇宙）

这是我的原始设计中缺失的最重要一层。制度是**显式可执行的社会规则**，而非隐含在 prompt 里。

```
制度类型：
├── 产权制度：资源归属权的定义与变更
├── 税收制度：资源征集的规则与比例
├── 司法制度：纠纷解决的程序
├── 治安制度：暴力行为的边界与惩罚
├── 贸易制度：交换的规则与限制
├── 继承制度：财富和地位如何传递
└── 外交制度：群体间互动的规则
```

制度如何涌现：
- 初期：自然状态（弱肉强食）
- 中期：非正式规范（习俗、声誉）
- 后期：正式制度（成文法律、组织章程）

制度如何改变：
- 外部压力触发（灾害、战争）
- 内部创新（技术进步、文化变化）
- 关键个体推动（L1 英雄智能体的制度设计）

### 3.4 社会世界层（生命宇宙）

```
个体身份：职业、性格、欲望、能力、健康、声望
关系网络：亲属、朋友、盟友、敌人、雇佣、组织
组织层级：家庭→部落→城邦→国家→帝国
宏观系统：市场、舆论、治理、外交、战争、教育、技术扩散
```

### 3.5 历史世界层（叙事宇宙）

这是让世界成为"文明"而非"NPC表演"的关键层。

- 每个城市有建城史
- 每个组织有创始事件、分裂、合并、覆灭
- 每条法律有修订链
- 每次战争有起因、过程、结果
- 每种文化有扩散路径
- 每个智能体有人生轨迹和社会位置变化

历史不仅记录，还驱动：
- 路径依赖（历史决策约束未来选项）
- 集体记忆（文化认同的基础）
- 因果追溯（任何现象可追溯到起源事件）

## 四、四层智能体系统

### 4.1 精确分层

| 层级 | 数量 | LLM调用 | 决策频率 | 典型角色 |
|------|------|---------|---------|---------|
| **L1 核心** | 50-80 | Opus, 全量 | 实时 | 城主、宗教领袖、叛军首领、科学家、外交官 |
| **L2 功能** | 500-1000 | Sonnet, 按需 | 每日-每周 | 工匠、商会成员、士兵、官员、教师 |
| **L3 背景** | 10,000-50,000 | Haiku, 极少 | 每周-每月 | 农民、矿工、渔夫、普通居民 |
| **L4 生态** | 100,000+ | 无 | 无 | 动物群、植被、微生物 |

### 4.2 跨层协议

```
升级触发条件（临时升格）：
- 智能体卷入冲突、交易、选举、审判
- 被系统检测为"历史关键节点"
- 进入重要组织核心
- 被用户摄像机聚焦
- 处于高激活区域（首都、边境、灾区）

降级规则：
- 事件处理完毕后降回原层级
- 高频升格降格自动调整基础层级
- L1 英雄可指定 L3 为代理人
```

### 4.3 感知半径

```
L1 核心：感知半径最大，可接收远距离重要信号
L2 功能：区域性感知，聚焦本地社区
L3 背景：最小感知半径，仅关注 immediate 周围
L4 生态：无感知，纯规则响应环境信号
```

## 五、Deliberation Router（决策路由）

这是项目最有创新性的工程组件之一。LLM 不是唯一大脑，而是高层认知器官。

### 5.1 算法选择矩阵

```
场景类型 → 推荐算法：

日常行动（生产/运输/采集）
  → Heuristic Planner (规则+评分函数)

资源调度（生产分配/库存管理）
  → Constraint Solver (优化求解)

行动探索（社交/探索新区域）
  → Contextual Bandit (探索/利用平衡)

长程规划（战争/扩张/技术路线）
  → MCTS / 搜索树

谈判/制度设计/战略决策
  → LLM Deliberation (反思+规划+协商)

紧急响应（灾害/袭击）
  → 规则优先，LLM 次之
```

### 5.2 路由决策

```
Deliberation Router 的输入：
- 当前场景类型
- 智能体层级
- 可用计算预算
- 当前世界状态（局部）
- 历史决策成本评估

Deliberation Router 的输出：
- 选择哪种算法
- 分配多少"思考预算"
- 是否请求组织意见（集体决策）
- 是否触发多智能体协商
```

### 5.3 热点升格机制

```
系统持续监控"热点区域"：
- 冲突区（暴力频率上升）
- 首都/政治中心
- 边境
- 灾区
- 经济枢纽（市场异常波动）

热点区域内的 L3 智能体自动获得更高的升格概率
避免全局扫描，仅对高价值区域做精细推理
```

```

│                                                      │

│     ↑            ↑            ↑              ↑        │
│  世界状态    传播衰减     智能体订阅表   决策执行     │

### 6.3 技术栈

| 模块 | 技术选型 | 理由 |
|------|---------|------|
| 主语言 | Python 3.11+ | LLM生态最成熟 |
| 异步框架 | asyncio + uvloop | 高性能事件循环 |
| LLM | Anthropic SDK | 成本最低，质量最高 |
| 本地缓存 | SQLite + LRU | 零依赖 |
| 向量存储 | Qdrant (本地) | 记忆检索 |
| 可视化 | Web (Tauri/WebGL) | 跨平台，图形能力强 |
| 数值计算 | NumPy/SciPy | 高性能数组运算 |
| 持久化 | SQLite + msgpack | 紧凑序列化 |

### 6.4 性能优化

```
Token 优化：
- 共享世界上下文（相邻智能体共用环境描述）
- 记忆压缩（定期摘要而非无限增长）
- 批量 API（多个请求打包）
- 缓存命中（相似场景结果复用）

计算优化：
- 空间分块（仅处理活跃区域）
- 异步分片（按地理分片并行）
- Lazy 激活（非活跃智能体延迟）
- 向量化（NumPy 处理批量数据）

内存优化：
- msgpack 序列化（比 JSON 小 3-5x）
- 记忆 LRU 淘汰
- 大地图 mmap 映射
```

## 七、学术价值体系（五大原创性创新）

### 创新一：去中心化感知-行动多智能体研究范式

**学术空白**：现有所有主流生成式智能体项目均采用中心化上帝视角，存在根本性的"仿真偏差"。

**原创性突破**：严格定义三条范式铁则，从"中心化仿真"走向"去中心化分布式仿真"。

**可支撑的研究**：
- 《分布式感知约束下多智能体系统的涌现性边界》(AAMAS/ICML/NeurIPS)
- 《局部信息不对称对群体决策的影响机制》

### 创新二：社会-生态双系统闭环耦合理论

**学术空白**：现有 AI 多智能体项目均将自然环境作为静态背景板，无法模拟文明-生态共演化。

**原创性突破**：首次在生成式多智能体系统中构建全量化、全闭环的社会-生态双向耦合模型。

**可支撑的研究**：
- 《资源丰度与文明兴衰的因果关系》(PNAS/Nature Sustainability)
- 《环境变化对社会制度演化的长期影响》

### 创新三：万级规模"算力分层涌现"方法

**学术空白**：每个智能体持续调用 LLM 导致算力呈指数增长，无法实现万级规模。

**原创性突破**：证明"复杂社会系统的宏观涌现，不需要每个个体都具备全量高阶认知能力"。

**可支撑的研究**：
- 《大规模生成式多智能体系统的算力-涌现性匹配模型》(NeurIPS/ICLR/MLSys)
- 《分层多智能体系统的创新扩散模型》

### 创新四：长周期文明演化多模态可解释基准数据集

**学术空白**：现有项目均为短周期（几天到几周）模拟，没有长周期（几百年到几千年）全链路数据。

**原创性突破**：构建全维度结构化数据记录系统，定义12项核心指标。

**可支撑的成果**：
- 发布基准数据集论文 (Scientific Data / NeurIPS Datasets Track)
- 成为多智能体研究领域的 ImageNet

### 创新五：去中心化多智能体系统的群体对齐与安全边界

**学术空白**：AI 对齐研究几乎全部集中在单智能体，群体对齐是完全的研究空白。

**可支撑的研究**：
- 《不同价值观初始设定对系统长期演化的影响》
- ICML SafeML Track / NeurIPS AI Safety Workshop / Nature Machine Intelligence

## 八、工程价值体系（三大核心创新）

### 创新一：事件驱动型分布式仿真引擎

**核心痛点**：帧同步中心化架构在万级规模下性能指数级下降。

**工程突破**：完全事件驱动的分布式引擎，无全局帧同步，万级智能体稳定性能。

**长期价值**：无限横向扩展能力，从笔记本无缝扩展到集群。

### 创新二：全模块化插件化引擎设计

**核心痛点**：现有项目高度耦合，无标准化接口，无法二次开发。

**工程突破**：定义行业级标准化接口规范，插件可替换任意模块。

**长期价值**：成为多智能体仿真领域的 Unity 引擎，全球开发者贡献插件。

### 创新三：零门槛全民科研平台

**核心痛点**：现有工具面向专业研究者，普通用户无法使用。

**工程突破**：Windows 原生一键启动 + 图形化配置界面 + 一键分享复现。

**长期价值**：推动多智能体研究的民主化，形成数据飞轮。

## 九、首发世界设定：河谷群岛文明

### 9.1 设定选择理由

现代城市太复杂，状态空间太大。河谷群岛世界：
- 世界直观，视频演示有吸引力
- 社会机制容易显式建模
- 生态-经济-制度-战争自然串联
- 学术上容易做控制变量实验

### 9.2 世界规格

```
地图：河谷群岛
- 约 512x512 格子
- 3-5 条主要河流
- 8-20 个岛屿/山谷聚落
- 森林覆盖率 40%
- 矿藏分布不均

初始人口：3000-8000 个智能体
